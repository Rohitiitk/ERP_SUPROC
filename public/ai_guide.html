AI Assistant Guide

Your guide to effectively using the AI Assistant to manage your ERP data.

How to Prompt the Assistant

To create a new item, state your intent clearly and provide all the required information for that record. The more specific you are, the better.

Example:

I need to create a work order to produce 200 units of the 'Industrial Power Supply'. Use BOM number BOM-IPS-004 and send the items to the 'Main Finished Goods' warehouse. Set the status to 'Planned', with a start date of 11-08-2025 and an end date of 22-08-2025.

To update a record, you must first identify it using a unique piece of information, like a Work Order number or an item name. Then, state what you want to change.

Example:

Update the status for the 'Industrial Power Supply' work order to 'In Progress' and set the actual start date to today.

Clearly state your intent to delete and identify the specific record you wish to remove. Deletions are permanent.

Example:

Please delete work order WO-2025-001 from the system.

E-commerce

Add a new product: 'Wireless Headphones', SKU 'HP-NC-2025', price is 149.99, and we have 250 in stock. Update the stock for SKU 'HP-NC-2025' to 235.

Education

Enroll student 'Jane Doe', ID 'S-JD-2025', into the 'Introduction to Python' course, section 'Fall-A'. Update the grade for student 'S-JD-2025' in 'Introduction to Python' to 'A-'.

Finance

Log a new transaction: a debit of 500.00 from account 'Main Checking' to 'Office Supplies'. Reconcile the transaction for 'Office Supplies' on August 1st.

Do's and Don'ts

Do

  • Be Specific: Provide all necessary details in one go.
  • Use Natural Language: Speak or type as you would to a person.
  • Refer to Records: Use names or IDs (like "Work Order WO-2025-001") to specify which item to update or delete.
  • Check Your Spelling: Ensure entity and field names are spelled correctly.

Don't

  • Be Vague: Avoid commands like "update the thing" or "fix it."
  • Ask to Change the UI: The AI cannot add buttons, change colors, or modify the application's design.
  • Combine Unrelated Tasks: Focus on one primary action (create, update, or delete) per command.
  • Assume It Remembers: Treat each new command as a fresh start. The AI has a very short-term memory.

Understanding the AI's Limitations

The AI Assistant is a powerful tool, but it's important to understand its boundaries to use it effectively.

  • It's a Language Translator: The AI's main job is to translate your natural language into a structured command that the database understands. It doesn't "think" or "reason" like a human.
  • It Can't See Your Screen: The AI has no knowledge of the user interface. It only knows about the entities and fields defined in the system's configuration.
  • It Can Misunderstand Ambiguity: If a command is unclear (e.g., "change the start date" without specifying *which* start date), it may ask for clarification or make an incorrect guess.
  • It Follows Its Rules: The AI can only perform the actions it was programmed for: creating, updating, deleting, and listing records. It cannot generate complex analytical reports beyond simple summaries or answer general knowledge questions.
