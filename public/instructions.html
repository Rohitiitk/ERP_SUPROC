Supabase Credentials Checklist

Follow these verified steps to collect the exact Supabase values required by Suproc: Project URL, Session Pooler connection string, and the Service Role key.

1

Sign in and open (or create) your Supabase project

All required credentials are stored inside the Supabase dashboard for a specific project. You can use an existing project or create a fresh one for Suproc.

  • ✓ Go to supabase.com/dashboard/projects and sign in with your GitHub or email login.
  • ✓ If you are spinning up a new backend, click Create new project, choose your organisation, pick a project name, region, and plan, then set a database password. This password is needed later for the Session Pooler connection string—store it securely.
  • ✓ If the project already exists, open it from the project list so the left-hand navigation is visible.
2

Confirm your database password

The Session Pooler connection string includes the Postgres password you set when the project was created. If you cannot recall it, Supabase allows you to reset it.

  1. In the left navigation, click Settings → Database.
  2. Under the General tab, locate the Database password panel.
  3. If you forgot the password, use the Reset database password action and store the new value. Resetting will require you to update any existing applications using the old password.
3

Copy the Supabase Project URL

This URL identifies your Supabase instance and is required by Suproc to reach the REST API.

  1. Go to Settings → API.
  2. At the top of the page, under Project Configuration, copy the value labelled Project URL.
  3. The URL typically looks like https://your-project-ref.supabase.co.

Example format

https://YOUR-PROJECT-REF.supabase.co
4

Copy the Service Role key

Suproc needs a key that can perform administrative actions on your Supabase tables. The service role key provides this access scope.

  1. Stay on Settings → API.
  2. Scroll to the Project API keys section.
  3. Click Reveal next to the service_role key and copy it.

Security reminder:

Treat the service role key like a password. Do not embed it in client-side code or share it outside your team. Suproc stores it encrypted for your workspace and uses it only for server-side automation.

5

Copy the Session Pooler connection string

Suproc connects to your Postgres database via Supabase’s connection pooler so it works reliably from environments without IPv6.

  1. Navigate to Settings → Database → Connection pooling.
  2. Select the Session tab (not Transaction).
  3. Copy the full connection string. It will resemble: postgresql://postgres:YOUR-DB-PASSWORD@YOUR-PROJECT-REF.pooler.supabase.com:6543/postgres

Replace YOUR-DB-PASSWORD with the password you verified in Step 2 before pasting into Suproc. Depending on your region, the host may end with .supabase.net or .supabase.com—copy exactly what Supabase shows.

6

Enter the values inside Suproc

With the Supabase credentials on hand, complete the Suproc ERP setup modal.

  1. In Suproc, click Start free workspace (or open the configuration modal from the ERP landing page).
  2. Populate the fields with the values you collected:
    • Supabase Project URL → paste the Project URL from Step 3.
    • Supabase DB URL (Session Pooler) → paste the connection string from Step 5, with your password substituted.
    • Supabase Service Role Key → paste the service_role key from Step 4.
  3. Click Save & Continue. Suproc will encrypt the credentials, run the initial schema setup, and confirm when the workspace is ready.

Keep the key and password in a secure vault. If you rotate them later, revisit this modal to update Suproc.
